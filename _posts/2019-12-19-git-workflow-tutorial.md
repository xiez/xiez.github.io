这是前阵子在组里做的一次分享，主要关于 Git 的日常使用和工作流程，目的是希望提高大家对 Git 的了解程度，从而提高工作效率。

之所以做这次分享，是因为看到大部分人只会使用 pull 和 push 来拉取和提交代码，并没有把 Git 真正的核心功能（分支）用起来。从而导致大量的时间用来应对各种问题，比如，代码丢失，大量冲突，无意中修改他人代码...
